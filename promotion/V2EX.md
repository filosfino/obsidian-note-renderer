# 做了个 Obsidian 插件，把 Markdown 笔记直接排成适合小红书发布的图片

最近我做了一个 Obsidian 插件：**Note Renderer**。

它解决的是一个很具体的问题：

平时很多内容本来就是先写在 Obsidian 里的，但如果想发到小红书，往往还要再折腾一遍排版，手动截图、贴背景、调字号、分段分页，流程比较碎。

所以我做了个插件，想把这件事尽量收敛成一件事：

**在 Obsidian 里写完笔记后，直接渲染成适合发布的分页图片。**

## 现在能做什么

- 按 `## 封面文字` / `## 封面图` / `## 正文` 的结构解析笔记
- 自动把正文分页，导出为多张 PNG
- 支持封面背景图和封面文字叠加
- 支持 `---` 手动分页
- 内置 9 个主题
- 支持长文（3:5）和图文（3:4）两种比例
- 支持 50+ 中文字体、字号调节、封面字体单独设置
- 支持描边、发光、色带、阴影等封面文字效果
- 支持遮罩、暗角、噪点、极光、散景等封面叠层效果
- 支持侧边栏实时预览、翻页、缩放
- 支持导出 ZIP
- 支持通过 Obsidian CLI 调用 `renderToFiles` API 做批量渲染

## 适合谁

- 本来就在 Obsidian 里写内容的人
- 想把笔记整理后发到小红书的人
- 想用 Markdown 而不是设计软件完成图文排版的人
- 想把内容生产和发布素材导出串起来的人

## 用法大概是这样

笔记结构示例：

```md
# 标题

## 封面文字
这是一篇想发布到小红书的内容

## 封面图
![[cover.jpg]]

## 正文
第一段内容。

第二段内容。

---

这里会强制分页。
```

写完以后，就可以在插件里预览并导出图片。

## 我比较在意的几个点

1. 不是把 Markdown 粗暴截图，而是按页面去重新排版。
2. 封面和正文是分开处理的，适合做“小红书封面 + 内页”这种结构。
3. 尽量保留写作者工作流，不需要把内容拷到别的工具里再做一次。
4. 除了 GUI，也支持 CLI，后面更方便接自动化流程。

- 用 Obsidian 写作
- 会把内容发到社媒平台
- 对 Markdown 到图片的渲染有需求

- 还有哪些排版能力是刚需
